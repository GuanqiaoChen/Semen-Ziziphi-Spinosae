# 酸枣仁八产地高光谱溯源流水线

`provenance_study/` 是 `main` 分支的主要科研实现。它只回答一个问题：在当前可用数据上，如何以可复算、防泄漏且结论有边界的方式，利用 392 波段高光谱对酸枣仁进行 HBS、HBX、HNA、HNX、NX、SXD、SXQ 和 XJH 八产地溯源。

完整的结果前方案见 [现有数据产地溯源算法探索与锁定验证方案](../docs/现有数据产地溯源算法探索与锁定验证方案.md)。

## 分组和盲态边界

每个产地的两张来源图像均按整数种子 ID 排序，再由固定算术子种子完成随机置换，以置换后次序对 10 取模。同一产地、批次索引相同的两份图像子集合并为一个构造批次。

- 批次 0–7：开发，1,012 粒；
- 批次 8–9：程序锁定内部测试，252 粒。

`discover_manifest()` 默认 `hash_files=False`，因此开发阶段只枚举锁定文件名、大小和路径，不打开其内容。`load_development_csv()` 在第一次 I/O 前硬拒绝任何锁定记录。只有带明确确认短语的正式评估入口才显式对全数据做 SHA-256 并读取批次 8–9。

## 冻结主方法

主预测器对逐粒平均光谱执行 SG 一阶导数（窗口 15、二阶多项式），随后对三个互补成员的概率等权平均：

1. 收缩 LDA；
2. `C=1` 的多类 LR；
3. `C=10` 的 RBF-SVM，使用开发批次 OOF `decision_function` 拟合唯一温度，不使用非分组 Platt 校准。

开发 OOF 中，三模型组合的 balanced accuracy 为 98.2413%，macro-F1 为 98.2396%，误分 18/1,012；最强单模型 SG1-收缩 LDA 为 97.5816%，误分 25/1,012。该 0.6598 个百分点差仅是开发证据，是否存在稳定实质增益由预先固定的锁定效应门槛决定。

稳定选带、逐粒 MAT 像素异质性/径向/形态视图、学习凸权重以及 321,776 参数的 1D 残差 CNN 均已在开发批次比较，未获得足以替代该简洁光谱组合的稳定证据。这些路线作为消融或负结果保留，不会在看到锁定结果后重新挑选。

## 执行顺序

在项目根目录使用本地虚拟环境：

```powershell
.venv\Scripts\python.exe -m unittest discover -s provenance_study/tests -v
.venv\Scripts\python.exe provenance_study/explore_development.py
.venv\Scripts\python.exe provenance_study/run_cnn_development.py
```

上述命令只允许读取开发批次。正式测试命令必须等待结果前代码、配置和方案提交后，再按入口 `--help` 显示的确认短语执行。不在本文档中给出可绕过守卫的快捷命令。

## 输出职责

- `outputs/development/`：严格 OOF 开发预测、模型比较和冻结温度；
- `outputs/development_cnn/`：内层早停、外层整批留出的 CNN 负结果；
- `outputs/locked_evaluation/`：一次性锁定预测、构造批次统计、效应门槛、错误和校准；
- `outputs/robustness/`：标签置换与两向整幅来源图像迁移；
- `figures/`：只从上述机器可读文件生成的 PNG/PDF 及绘图源表。

## 解释边界

构造批次是经研究团队授权的当前数据内部分组单位，但它们仍共享每产地的两张来源图像。因此，本流水线评估的是当前数据内部的八产地判别能力，不是新农场、新年份、新设备或未知产地的外部认证。两向整幅图像迁移专门用来暴露这一采集域局限，不会被写成替代研究目标的主结果。
